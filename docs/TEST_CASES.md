## 一、待办事项管理（TC-001 ~ TC-005）

### TC-001: 创建新待办

- **模块**: 待办事项管理
- **优先级**: P0
- **前置条件**: 应用已启动，主窗口处于可操作状态，至少存在一个可用标签
- **测试步骤**:
  1. 在待办输入区域的标题输入框中输入待办标题，例如「完成项目周报」
  2. 点击截止日期选择器，选择一个未来的日期和时间
  3. 在标签选择区域中选择一个或多个已有标签
  4. 点击「添加」按钮提交待办
- **预期结果**:
  - 待办列表中出现新创建的待办项
  - 待办标题、截止日期、标签信息均正确显示
  - 新待办的默认状态为「未开始」
  - 输入区域在提交后自动清空，准备接收下一条待办
- **适用版本**: 两者

---

### TC-002: 编辑待办

- **模块**: 待办事项管理
- **优先级**: P0
- **前置条件**: 待办列表中已存在至少一条待办记录
- **测试步骤**:
  1. 在待办列表中找到目标待办，点击该待办项进入编辑模式（或点击编辑按钮）
  2. 修改待办标题为新内容，例如将「完成项目周报」改为「完成项目月报」
  3. 修改截止日期为一个新的日期
  4. 确认修改（点击保存按钮或按下回车键）
- **预期结果**:
  - 待办列表中对应条目的标题已更新为修改后的内容
  - 截止日期已更新为新设置的日期
  - 其他未修改的字段（如标签、状态）保持不变
  - 编辑模式正常退出，恢复到列表展示状态
- **适用版本**: 两者

---

### TC-003: 删除待办

- **模块**: 待办事项管理
- **优先级**: P0
- **前置条件**: 待办列表中已存在至少一条待办记录，记录当前列表总数
- **测试步骤**:
  1. 在待办列表中找到目标待办项
  2. 点击该待办项的删除按钮
  3. 在弹出的确认对话框中点击「确认」删除
- **预期结果**:
  - 被删除的待办项从列表中消失
  - 列表总数减少一条
  - 删除操作不影响其他待办项的显示和数据
  - 若删除后列表为空，显示空状态提示
- **适用版本**: 两者

---

### TC-004: 切换待办状态

- **模块**: 待办事项管理
- **优先级**: P0
- **前置条件**: 待办列表中已存在一条状态为「未开始」的待办
- **测试步骤**:
  1. 找到状态为「未开始」的待办项，点击状态切换按钮或状态标识区域
  2. 观察状态变化，确认状态切换为「进行中」
  3. 再次点击状态切换按钮
  4. 观察状态变化，确认状态切换为「已完成」
- **预期结果**:
  - 状态按照「未开始 → 进行中 → 已完成」的顺序正确流转
  - 每次状态切换后，界面上的状态标识（图标/文字/颜色）立即更新
  - 「已完成」状态的待办项呈现视觉区分效果（如删除线、灰色显示等）
  - 状态切换后数据持久化保存，刷新页面后状态不丢失
- **适用版本**: 两者

---

### TC-005: 筛选和排序

- **模块**: 待办事项管理
- **优先级**: P1
- **前置条件**: 待办列表中存在多条不同状态、不同标签、不同截止日期的待办
- **测试步骤**:
  1. 点击筛选栏中的状态筛选按钮，选择「进行中」，观察列表变化
  2. 清除状态筛选，切换到标签筛选，选择某一特定标签，观察列表变化
  3. 清除标签筛选，切换到日期筛选（如「今天到期」「本周到期」），观察列表变化
  4. 清除所有筛选条件，点击排序选项，按「截止日期」升序排列
  5. 切换排序方式为按「创建时间」降序排列
- **预期结果**:
  - 按状态筛选后，列表仅显示符合所选状态的待办
  - 按标签筛选后，列表仅显示包含所选标签的待办
  - 按日期筛选后，列表仅显示符合日期范围的待办
  - 按截止日期排序后，列表按截止日期从近到远排列
  - 按创建时间排序后，列表按创建时间从新到旧排列
  - 筛选条件之间可组合使用，结果取交集
- **适用版本**: 两者

## 二、Widget 浮层功能（TC-006 ~ TC-010）

### TC-006: Widget 自动弹出

- **模块**: Widget 浮层功能
- **优先级**: P0
- **前置条件**: 应用已启动，主窗口当前可见，Widget 当前未显示
- **测试步骤**:
  1. 关闭主窗口（点击窗口标题栏的关闭按钮）
  2. 等待 1~2 秒，观察桌面变化
- **预期结果**:
  - 主窗口关闭后（最小化到托盘），Widget 浮层自动出现在桌面上
  - Widget 浮层显示在合理的默认位置（如屏幕右下角）
  - Widget 浮层始终保持在其他窗口之上（置顶显示）
  - Widget 浮层正确显示待办摘要信息
- **适用版本**: 两者

---

### TC-007: Widget 显示待办列表

- **模块**: Widget 浮层功能
- **优先级**: P1
- **前置条件**: Widget 浮层已显示，待办列表中存在多条不同截止日期的待办
- **测试步骤**:
  1. 观察 Widget 浮层中的待办列表内容
  2. 核对 Widget 中显示的待办项与主窗口中的数据是否一致
  3. 检查待办项的排序方式
- **预期结果**:
  - Widget 按截止日期由近到远的顺序显示待办项
  - 最近即将到期的待办排在最前面
  - 每条待办至少显示标题和截止日期信息
  - 已逾期的待办有明显的视觉标识（如红色高亮）
  - Widget 中的数据与主窗口完全同步
- **适用版本**: 两者

---

### TC-008: Widget 快速添加待办

- **模块**: Widget 浮层功能
- **优先级**: P1
- **前置条件**: Widget 浮层已显示
- **测试步骤**:
  1. 在 Widget 浮层中找到快速添加输入框
  2. 输入待办标题，例如「买菜」
  3. 按下回车键或点击添加按钮提交
  4. 打开主窗口，检查待办列表
- **预期结果**:
  - Widget 中成功添加待办，输入框清空
  - Widget 待办列表中立即显示新增的待办
  - 主窗口中的待办列表同步显示新增条目
  - 通过 Widget 添加的待办默认状态为「未开始」
- **适用版本**: 两者

---

### TC-009: Widget 完成待办

- **模块**: Widget 浮层功能
- **优先级**: P1
- **前置条件**: Widget 浮层已显示，且 Widget 中至少有一条未完成的待办
- **测试步骤**:
  1. 在 Widget 中找到一条未完成的待办项
  2. 点击该待办项的完成按钮（复选框或完成图标）
  3. 打开主窗口，检查该待办的状态
- **预期结果**:
  - Widget 中该待办项的状态更新为「已完成」，呈现完成态视觉效果
  - 主窗口中对应待办的状态同步更新为「已完成」
  - Widget 列表根据排序规则重新排列（已完成项可能移至底部或隐藏）
- **适用版本**: 两者

---

### TC-010: Widget 拖拽移动

- **模块**: Widget 浮层功能
- **优先级**: P2
- **前置条件**: Widget 浮层已显示在桌面上，记录当前 Widget 位置
- **测试步骤**:
  1. 将鼠标移动到 Widget 浮层的标题栏或可拖拽区域
  2. 按住鼠标左键不放，向桌面其他位置拖动
  3. 释放鼠标左键
  4. 关闭应用后重新启动，观察 Widget 位置
- **预期结果**:
  - Widget 浮层跟随鼠标移动到新位置
  - 拖拽过程流畅，无明显卡顿或闪烁
  - 释放鼠标后 Widget 停留在释放位置
  - 重新启动应用后，Widget 记住上次的位置（位置持久化）
  - Widget 不能被拖出屏幕可视区域之外
- **适用版本**: 两者

## 三、主题切换（TC-011 ~ TC-012）

### TC-011: 深色/浅色主题切换

- **模块**: 主题切换
- **优先级**: P1
- **前置条件**: 应用已启动，当前为浅色主题（默认主题）
- **测试步骤**:
  1. 找到主题切换按钮（通常在标题栏或设置区域）
  2. 点击切换为深色主题
  3. 逐一检查以下组件的样式变化：标题栏、侧边栏、待办列表、输入框、按钮、筛选栏、滚动条
  4. 再次点击主题切换按钮，切换回浅色主题
  5. 关闭应用后重新启动
- **预期结果**:
  - 切换为深色主题后，所有组件的背景色、文字色、边框色等正确更新
  - 深色主题下文字具有良好的可读性，对比度符合无障碍标准
  - 切换过程平滑，无明显的白屏闪烁
  - 切换回浅色主题后，所有组件恢复原样
  - 重新启动后，应用保持上次选择的主题设置（主题持久化）
- **适用版本**: 两者

---

### TC-012: Widget 主题同步

- **模块**: 主题切换
- **优先级**: P1
- **前置条件**: 应用已启动，Widget 浮层已显示，当前为浅色主题
- **测试步骤**:
  1. 确认 Widget 浮层当前为浅色主题样式
  2. 在主窗口中将主题切换为深色主题
  3. 立即观察 Widget 浮层的样式变化
  4. 在主窗口中将主题切换回浅色主题
  5. 再次观察 Widget 浮层的样式变化
- **预期结果**:
  - 主窗口切换为深色主题后，Widget 浮层同步切换为深色主题
  - Widget 的背景色、文字色、边框色等与主窗口风格保持一致
  - 主题同步过程即时生效，无明显延迟
  - 切换回浅色主题后，Widget 同步恢复为浅色主题
- **适用版本**: 两者

## 四、系统托盘（TC-013 ~ TC-016）

- **预期结果**:
  - 系统托盘区域出现应用图标
  - 托盘图标清晰可辨，与应用图标一致
  - 鼠标悬停在托盘图标上时显示应用名称的提示文字（Tooltip）
- **适用版本**: Electron

---

### TC-014: 托盘右键菜单

- **模块**: 系统托盘
- **优先级**: P0
- **前置条件**: 应用已最小化到系统托盘，托盘图标可见
- **测试步骤**:
  1. 右键点击系统托盘中的应用图标
  2. 观察弹出的上下文菜单内容
  3. 点击「显示/隐藏窗口」菜单项，验证主窗口显示切换
  4. 再次右键点击托盘图标，点击「显示/隐藏 Widget」菜单项
  5. 再次右键点击托盘图标，点击「退出」菜单项
- **预期结果**:
  - 右键点击后弹出上下文菜单，包含以下选项：「显示/隐藏窗口」「显示/隐藏 Widget」「退出」
  - 点击「显示/隐藏窗口」后，主窗口在显示和隐藏之间切换
  - 点击「显示/隐藏 Widget」后，Widget 浮层在显示和隐藏之间切换
  - 点击「退出」后，应用完全关闭（主窗口、Widget、托盘图标均消失），进程退出
- **适用版本**: Electron

  - 主窗口获得焦点并显示在最前面
  - 窗口恢复到最小化之前的尺寸和位置
  - 窗口内的数据和状态保持不变

### TC-016: 关闭行为设置

- **模块**: 系统托盘
- **优先级**: P1
- **前置条件**: 应用已启动，主窗口可见
- **测试步骤**:
  1. 在应用设置中找到「关闭行为」配置项
  2. 设置为「最小化到托盘」，点击主窗口的关闭按钮，观察行为
  3. 从托盘恢复窗口，将关闭行为改为「直接退出」，再次点击关闭按钮
  4. 重新启动应用，将关闭行为改为「每次询问」，点击关闭按钮
- **预期结果**:
  - 设置为「最小化到托盘」时：点击关闭按钮后窗口隐藏，应用继续在托盘运行
  - 设置为「直接退出」时：点击关闭按钮后应用完全退出，进程结束
  - 设置为「每次询问」时：点击关闭按钮后弹出对话框，提供「最小化到托盘」和「退出」两个选项
  - 关闭行为设置在应用重启后保持生效（配置持久化）
- **适用版本**: Electron

## 五、截止日期提醒（TC-017 ~ TC-018）

### TC-017: 24小时内到期提醒

- **模块**: 截止日期提醒
- **优先级**: P0
- **前置条件**: 应用已启动且在后台运行，存在一条截止日期在当前时间 24 小时以内的待办
- **测试步骤**:
  1. 创建或修改一条待办，将截止日期设置为当前时间之后 1 小时内
  2. 等待系统触发提醒检查周期（或手动触发提醒检查）
  3. 观察系统通知区域
- **预期结果**:
  - 系统弹出桌面通知（Toast / Notification），提示待办即将到期
  - 通知内容包含待办标题和截止时间
  - 通知图标为应用图标
  - 点击通知后打开主窗口并聚焦到对应待办
  - 同一条待办不会重复发送提醒通知
- **适用版本**: Electron

---

### TC-018: 已逾期标识

- **模块**: 截止日期提醒
- **优先级**: P0
- **前置条件**: 待办列表中存在一条截止日期已过期的未完成待办
- **测试步骤**:
  1. 创建一条待办，将截止日期设置为过去的某个时间
  2. 观察该待办在列表中的显示样式
  3. 检查 Widget 浮层中该待办的显示样式
- **预期结果**:
  - 逾期待办在主窗口列表中显示红色标记或红色高亮提示
  - 逾期标识明显区别于正常待办，易于识别
  - Widget 浮层中逾期待办同样显示红色标记
  - 逾期标识仅对未完成状态的待办生效，已完成的待办不显示逾期标记
- **适用版本**: 两者

## 六、标签管理（TC-019 ~ TC-021）

### TC-019: 创建标签

- **模块**: 标签管理
- **优先级**: P0
- **前置条件**: 应用已启动，标签管理界面可访问
- **测试步骤**:
  1. 打开标签管理界面（或在创建待办时进入标签管理入口）
  2. 点击「新建标签」按钮
  3. 输入标签名称，例如「工作」
  4. 选择一个标签颜色（从预设颜色盘中选择，例如蓝色）
  5. 点击「确认」或「保存」按钮
- **预期结果**:
  - 标签列表中出现新创建的标签
  - 标签名称和颜色与输入的一致
  - 新标签可在创建/编辑待办时作为可选项使用
  - 不允许创建名称为空的标签，提交时给出相应提示
  - 不允许创建与已有标签同名的标签，提交时给出相应提示
- **适用版本**: 两者

---

### TC-020: 编辑标签

- **模块**: 标签管理
- **优先级**: P1
- **前置条件**: 已存在至少一个标签，且该标签已被一条或多条待办引用
- **测试步骤**:
  1. 打开标签管理界面
  2. 找到目标标签，点击编辑按钮
  3. 修改标签名称为新名称，例如将「工作」改为「办公」
  4. 修改标签颜色为新颜色，例如从蓝色改为绿色
  5. 保存修改
  6. 返回待办列表，检查引用该标签的所有待办
- **预期结果**:
  - 标签列表中对应标签的名称和颜色已更新
  - 所有引用该标签的待办项中，标签名称和颜色同步更新
  - Widget 浮层中引用该标签的待办显示更新后的标签信息
  - 筛选栏中的标签选项同步更新为新名称和新颜色
- **适用版本**: 两者

### TC-021: 删除标签

- **模块**: 标签管理
- **优先级**: P1
- **前置条件**: 已存在至少一个标签，且该标签已被一条或多条待办引用
- **测试步骤**:
  1. 打开标签管理界面
  2. 找到目标标签，点击删除按钮
  3. 在弹出的确认对话框中确认删除
  4. 返回待办列表，检查之前引用该标签的待办
  5. 检查筛选栏中的标签选项列表
- **预期结果**:
  - 标签从标签管理列表中移除
  - 之前引用该标签的所有待办中，该标签引用被自动移除
  - 待办的其他信息（标题、截止日期、状态、其他标签）不受影响
  - 筛选栏中不再出现被删除的标签选项
  - 删除操作不可逆，确认对话框中有明确的警告提示
- **适用版本**: 两者
