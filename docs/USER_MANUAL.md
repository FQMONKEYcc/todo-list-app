## 1. 快速开始

### 1.1 系统要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | Windows 10 及以上（64 位） |
| 磁盘空间 | Electron 版本约 110 MB；Neutralinojs 版本约 3 MB |
| 运行时依赖 | 无需额外安装运行时环境，两个版本均为独立可执行文件 |

### 1.2 版本选择指南

本应用提供两个独立版本，功能界面完全一致，底层技术方案不同。请根据自身需求选择：

| 对比维度 | Electron 版本 | Neutralinojs 版本 |
|----------|---------------|-------------------|
| 安装包大小 | ~110 MB | ~3 MB |
| 底层框架 | Electron 33 (Chromium + Node.js) | Neutralinojs v6.5 (系统 WebView) |
| 系统通知 | 原生支持（截止日期到期自动推送） | 通过系统 API 支持 |
| Widget 浮层 | 独立无边框窗口，支持拖拽 | 独立窗口实现 |
| 系统托盘 | 完整支持（右键菜单、双击恢复） | 支持（Windows/Linux） |
| 适用场景 | 追求完整桌面体验，不在意体积 | 追求极致轻量，快速部署 |

**建议：**
- 如果您的磁盘空间充裕且希望获得最完整的桌面通知体验，选择 **Electron 版本**。
- 如果您希望应用体积尽可能小、启动迅速，选择 **Neutralinojs 版本**。

### 1.3 安装与启动

#### Electron 版本

1. 从 `release` 目录获取 `TodoList.exe`（便携版，无需安装）。
2. 将 `TodoList.exe` 放置到您希望的目录中。
3. 双击 `TodoList.exe` 即可启动应用。

> 首次启动时，应用会自动在系统用户数据目录下创建 `todos.json` 文件用于存储数据。

#### Neutralinojs 版本

1. 从 `neutralino-version/dist` 目录获取发行包。
2. 运行 `TodoList.exe`（Windows）即可启动。

> Neutralinojs 版本会在系统 `AppData` 目录的 `TodoList` 文件夹下创建 `todos.json` 文件。

#### 从源码运行（开发者）

**Electron 版本：**

```bash
cd electron-version
npm install
npm start
```

**Neutralinojs 版本：**

```bash
cd neutralino-version
# 直接运行已编译的二进制文件
./bin/neutralino-win_x64.exe
```

## 2. 核心功能

### 2.1 待办管理（CRUD）

#### 新建待办

1. 点击界面顶部的 **"New Todo"** 按钮，或使用快捷键 `Ctrl+N`。
2. 在弹出的模态框中填写以下信息：
   - **标题**（必填）：待办事项的名称。
   - **描述**（可选）：对待办事项的详细说明。
   - **截止日期**（可选）：设定截止时间，用于提醒和排序。
   - **状态**：默认为"未开始"，也可在创建时直接选择其他状态。
   - **标签**（可选）：点击标签色块进行选择，支持多选。
3. 点击 **"Save"** 保存，或点击 **"Cancel"** / 按 `Esc` 取消。

> 保存成功后，模态框会出现短暂的成功闪烁动画反馈。

#### 编辑待办

- 在待办列表中 **点击任意待办卡片**，即可打开编辑模态框。
- 也可以点击卡片右侧的 **编辑图标**（铅笔图标）进入编辑。
- 修改完成后点击 **"Save"** 保存。

#### 删除待办

1. 点击待办卡片右侧的 **删除图标**（垃圾桶图标）。
2. 系统弹出确认对话框：`Delete "待办标题"?`
3. 点击 **确定** 执行删除，点击 **取消** 放弃操作。

> 删除操作不可撤销，请谨慎操作。

### 2.2 状态流转

每个待办事项有三种状态，按以下顺序循环切换：

```
未开始 --> 进行中 --> 已完成 --> 未开始（循环）
```

**切换方式：**
- 点击待办卡片左侧的 **状态圆点按钮**，即可将状态推进到下一阶段。
- 也可以在编辑模态框中通过下拉菜单直接选择目标状态。

**状态说明：**

| 状态 | 含义 | 视觉标识 |
|------|------|----------|
| 未开始 | 待办尚未开始处理 | 空心圆点 |
| 进行中 | 待办正在进行中 | 半填充圆点 |
| 已完成 | 待办已经完成 | 实心圆点（带勾选效果） |

> 当待办状态变为"已完成"时，系统会自动记录完成时间（`completedAt`）。若状态从"已完成"切换回其他状态，完成时间会被清除。

### 2.3 标签管理

标签系统用于对待办事项进行分类，每个标签包含名称和颜色两个属性。

#### 预置标签

应用初始提供三个默认标签：

| 标签名 | 颜色 |
|--------|------|
| Work | 蓝色 (#4a90d9) |
| Personal | 绿色 (#27ae60) |
| Urgent | 红色 (#e74c3c) |

#### 管理标签

1. 点击界面上的 **"Manage Tags"** 按钮，打开标签管理模态框。
2. 在模态框中可以执行以下操作：
   - **新建标签**：在底部输入框填写标签名称，选择颜色，点击 **"Add"** 按钮。
   - **编辑标签名称**：直接修改标签名称输入框的内容，失焦后自动保存。
   - **修改标签颜色**：点击标签左侧的颜色选择器，选择新颜色后自动保存。
   - **删除标签**：点击标签右侧的 **x** 按钮。如果该标签已被待办引用，系统会提示受影响的待办数量，确认后标签将从所有待办中移除。

#### 在待办中使用标签

- 在新建或编辑待办的模态框中，点击标签色块即可选中/取消选中。
- 支持同时选择多个标签。
- 模态框内还提供 **快速添加标签** 功能：输入名称和颜色后点击添加，新标签会自动被选中。

### 2.4 筛选与排序

#### 状态筛选

界面上方提供四个状态筛选按钮，每个按钮旁显示对应数量：

| 按钮 | 功能 |
|------|------|
| All | 显示所有待办 |
| 未开始 | 仅显示"未开始"状态的待办 |
| 进行中 | 仅显示"进行中"状态的待办 |
| 已完成 | 仅显示"已完成"状态的待办 |

#### 标签筛选

- 点击筛选区域的标签按钮可按标签筛选待办。
- 支持多选，多个标签之间为 **OR（或）** 逻辑：只要待办包含任意一个已选标签即会显示。

#### 日期范围筛选

通过日期选项卡可以快速缩小查看范围：

| 选项 | 说明 |
|------|------|
| All | 显示所有日期的待办 |
| Today | 仅显示今日创建或今日截止的待办 |
| This Week | 仅显示本周（周一至周日）创建或截止的待办 |

#### 关键词搜索

- 使用搜索框（快捷键 `Ctrl+F`）输入关键词。
- 搜索范围覆盖待办的 **标题** 和 **描述** 字段。
- 搜索为实时筛选（带 200ms 防抖），输入即时生效。

#### 排序

支持以下排序方式，通过下拉菜单选择：

| 排序依据 | 说明 |
|----------|------|
| 截止日期 (deadline) | 按截止时间排序，无截止日期的排在最后 |
| 创建时间 (createdAt) | 按创建时间排序 |
| 状态 (status) | 按状态排序（未开始 > 进行中 > 已完成） |

点击排序按钮旁的 **方向箭头** 可切换升序/降序。

> 所有筛选和排序偏好会自动保存，下次打开应用时自动恢复。

## 3. 高级功能

### 3.1 Widget 浮层

Widget 是一个轻量级悬浮窗口，用于在主窗口最小化或隐藏时快速查看和操作待办事项。

#### 触发方式

Widget 在以下情况自动出现：

- 主窗口 **最小化** 时。
- 主窗口 **关闭并选择"最小化到托盘"** 时。

当主窗口恢复显示时，Widget 自动隐藏。

#### Widget 功能

| 功能 | 说明 |
|------|------|
| 查看即将到期的待办 | 自动显示最近截止的 20 条未完成待办，按截止日期排序 |
| 快速完成待办 | 点击待办左侧的圆形按钮，将其标记为"已完成" |
| 快速添加待办 | 点击 **+** 按钮展开表单，输入标题和截止日期后提交 |
| 自动刷新 | 每 60 秒自动更新待办列表 |
| 关闭 Widget | 右键点击 Widget 区域即可关闭 |

#### Widget 中的截止日期显示

| 显示 | 含义 |
|------|------|
| `Overdue Xh` / `Overdue Xd` | 已超期 X 小时/天 |
| `Xh left` | 距截止还有 X 小时 |
| `Xd left` | 距截止还有 X 天 |
| `M/D HH:MM` | 超过 7 天时显示具体日期时间 |

#### Widget 位置

Widget 默认出现在屏幕 **右下角**（距边缘 20 像素），大小为 320x360 像素，无边框、透明背景、不在任务栏显示。

### 3.2 主题切换

应用支持 **深色模式** 和 **浅色模式** 两种主题。

- 点击界面右上角的 **主题切换按钮**（太阳/月亮图标）进行切换。
- 切换时有旋转缩放动画效果。
- 默认主题为 **深色模式**。
- 主题偏好自动保存，下次启动时自动应用。

### 3.3 系统托盘

#### Electron 版本

应用启动后会在系统托盘区域显示图标，提供以下操作：

| 操作 | 说明 |
|------|------|
| 双击托盘图标 | 恢复显示主窗口并获得焦点 |
| 右键点击托盘图标 | 打开托盘菜单 |

**托盘右键菜单选项：**

| 菜单项 | 功能 |
|--------|------|
| Show Window | 显示主窗口 |
| New Todo | 显示主窗口并打开新建待办表单 |
| Quit | 完全退出应用 |

#### 窗口关闭行为

首次点击窗口关闭按钮时，应用会弹出询问对话框：

| 选项 | 说明 |
|------|------|
| Minimize to Tray | 隐藏到托盘，Widget 浮层自动出现 |
| Quit | 完全退出应用 |
| Cancel | 取消关闭操作 |

勾选 **"Don't ask again"** 可以记住您的选择，后续关闭时将直接执行上次选择的行为。

> 如需重置关闭行为偏好，可以编辑 `todos.json` 文件中 `settings.closeAction` 字段，将其改回 `"ask"`。

#### Neutralinojs 版本

- 系统托盘同样可用（Windows 和 Linux 平台）。
- 托盘菜单提供 **版本信息查看** 和 **退出** 功能。

### 3.4 截止日期提醒

**此功能仅在 Electron 版本中可用。**

应用每 60 秒自动检查一次所有未完成待办的截止日期，并根据紧急程度推送系统通知：

| 时间范围 | 通知标题 | 提醒频率 |
|----------|----------|----------|
| 截止时间已过 | Deadline Reached! | 首次到期立即通知，之后每 30 分钟重复 |
| 30 分钟以内 | Due Soon | 每 15 分钟通知一次 |
| 30 分钟 ~ 1 小时 | Upcoming Deadline | 仅通知一次 |

**通知交互：**
- 点击系统通知会 **自动打开主窗口**，清除所有筛选条件，并 **高亮定位** 到对应的待办卡片（附带 2 秒的高亮边框动画）。

> 已超期的待办会显示超期时长（分钟/小时/天），提醒您尽快处理。

### 3.5 快捷键列表

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + N` | 新建待办 |
| `Ctrl + F` | 聚焦搜索框（自动全选已有内容） |
| `Esc` | 关闭当前打开的模态框（新建/编辑待办、标签管理） |
| `Enter` | 在标签名称输入框中按下可快速添加标签 |
| `Enter` | 在 Widget 快速添加表单中按下可提交待办 |

## 4. 数据管理

### 4.1 数据存储位置

所有待办数据、标签和设置保存在一个 JSON 文件中：`todos.json`。

#### Electron 版本

数据存储在 Electron 的 `userData` 目录下：

```
Windows: C:\Users\<用户名>\AppData\Roaming\todolist\todos.json
```

> 具体路径取决于 `package.json` 中 `name` 字段的值（当前为 `todolist`）。

#### Neutralinojs 版本

数据存储在系统数据目录的 `TodoList` 子文件夹下：

```
Windows: C:\Users\<用户名>\AppData\Roaming\TodoList\todos.json
```

> Neutralinojs 版本通过 `Neutralino.os.getPath('data')` 获取系统数据目录。

#### 数据文件结构

`todos.json` 文件的顶层结构如下：

```json
{
  "version": 1,
  "tags": [
    { "id": "tag_default_1", "name": "Work", "color": "#4a90d9" }
  ],
  "todos": [
    {
      "id": "todo_1234567890_abc12",
      "title": "待办标题",
      "description": "待办描述",
      "status": "未开始",
      "deadline": "2026-03-15T10:00:00.000Z",
      "tagIds": ["tag_default_1"],
      "createdAt": "2026-03-01T08:00:00.000Z",
      "updatedAt": "2026-03-01T08:00:00.000Z",
      "completedAt": null,
      "notifiedAt": null
    }
  ],
  "settings": {
    "theme": "dark",
    "sortBy": "deadline",
    "sortOrder": "asc",
    "filterStatus": "all",
    "filterTagIds": [],
    "closeAction": "ask",
    "filterDateRange": "all"
  }
}
```

### 4.2 数据备份

由于数据以单个 JSON 文件存储，备份非常简单：

1. **手动备份**：找到 `todos.json` 文件（路径见上方），复制到安全位置即可。
2. **定期备份建议**：可以将 `todos.json` 所在文件夹加入云同步服务（如 OneDrive、坚果云等）。

**备份步骤示例：**

```bash
# 打开数据目录（Electron 版本）
explorer "%APPDATA%\todolist"

# 打开数据目录（Neutralinojs 版本）
explorer "%APPDATA%\TodoList"

# 复制文件到备份位置
copy "%APPDATA%\todolist\todos.json" "D:\Backup\todos_backup_%date%.json"
```

> 应用采用 **先写临时文件再重命名** 的安全写入策略（Electron 版本），避免写入中断导致数据损坏。

### 4.3 数据迁移

#### 在两个版本之间迁移

Electron 版本和 Neutralinojs 版本使用完全相同的 `todos.json` 数据格式，因此可以直接互相迁移：

1. 关闭当前正在运行的应用。
2. 将源版本的 `todos.json` 复制到目标版本的数据目录下。
3. 启动目标版本的应用，数据将自动加载。

#### 迁移到新电脑

1. 在旧电脑上找到并复制 `todos.json` 文件。
2. 在新电脑上安装并首次运行应用（此时会生成默认数据文件）。
3. 关闭应用。
4. 用旧电脑的 `todos.json` 覆盖新电脑上的同名文件。
5. 重新启动应用。

> 迁移前建议先备份目标位置的现有数据文件。

## 5. 故障排除

### 5.1 常见问题（FAQ）

#### Q: 应用启动后显示空白窗口

**可能原因及解决方法：**
- 数据文件损坏。尝试删除 `todos.json`，应用会在下次启动时自动创建默认数据文件（注意：这会丢失所有现有数据，请先备份）。
- Electron 版本：确保运行环境为 Windows 10 或更高版本的 64 位系统。

#### Q: 关闭窗口后找不到应用了

**原因：** 您之前选择了"Minimize to Tray"并勾选了"Don't ask again"。

**解决方法：**
- 查看系统托盘区域（任务栏右下角，可能需要点击展开箭头），双击 Todo List 图标恢复窗口。
- 如果想恢复每次询问的行为，编辑 `todos.json`，将 `settings.closeAction` 改为 `"ask"`。

#### Q: 没有收到截止日期通知

**排查步骤：**
1. 确认使用的是 **Electron 版本**（Neutralinojs 版本的通知机制不同）。
2. 检查 Windows 通知设置：`Windows 设置 > 系统 > 通知`，确保应用通知未被关闭。
3. 确认待办设置了截止日期且状态不是"已完成"。
4. 通知触发时机：距截止 1 小时内才会开始推送。

#### Q: Widget 浮层没有出现

**可能原因：**
- Widget 仅在主窗口最小化或隐藏到托盘时出现。
- 确认主窗口不在显示状态——显示主窗口时 Widget 会自动隐藏。
- Widget 出现在屏幕右下角，检查是否被其他窗口遮挡。

#### Q: 数据丢失了

**预防措施：**
- 应用使用安全写入策略（先写 `.tmp` 临时文件再重命名），正常使用下不会丢失数据。
- 如果确实发生了数据丢失，检查数据目录下是否有 `todos.json.tmp` 文件，该文件可能包含最后一次保存的数据。

**恢复步骤：**
1. 如果 `todos.json.tmp` 存在，将其重命名为 `todos.json`。
2. 如果有备份文件，用备份文件替换当前文件。

#### Q: 如何完全退出应用

- **方法一**：右键点击系统托盘图标，选择 **Quit**。
- **方法二**：在关闭窗口的询问对话框中选择 **Quit**。
- **方法三**：通过任务管理器结束进程。

#### Q: 如何重置所有设置

将 `todos.json` 中的 `settings` 字段恢复为默认值：

```json
{
  "theme": "dark",
  "sortBy": "deadline",
  "sortOrder": "asc",
  "filterStatus": "all",
  "filterTagIds": [],
  "closeAction": "ask",
  "filterDateRange": "all"
}
```

或者直接删除 `todos.json` 文件，应用下次启动时会重新生成包含默认设置和默认标签的数据文件（所有待办将丢失）。

### 5.2 已知限制

| 限制项 | 说明 |
|--------|------|
| 平台支持 | 当前构建仅针对 Windows 64 位。Neutralinojs 版本二进制文件支持多平台，但未提供 macOS/Linux 的完整打包 |
| 数据同步 | 不支持多设备实时同步，数据仅存储在本地文件中 |
| 并发访问 | 同一数据文件不支持多实例同时写入，请勿同时运行同一数据目录的多个应用实例 |
| 撤销功能 | 删除待办和删除标签操作不可撤销 |
| 附件支持 | 不支持为待办添加文件附件 |
| 导出功能 | 不提供 CSV/Excel 等格式的导出功能，数据仅以 JSON 格式存储 |
| macOS 托盘 | Neutralinojs 版本在 macOS 上的系统托盘功能暂不可用（上游已知问题） |
| 通知限制 | 截止日期通知功能仅在 Electron 版本中完整支持 |
| 标题长度 | Widget 快速添加的待办标题最长 500 个字符 |
